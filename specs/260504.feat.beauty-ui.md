## 现状分析

### 1. Demo 布局 (src/demo/App.tsx)

- 外层容器使用 `flex gap-4 items-start`，时间轴固定 `w-[800px]`
- 没有设置整体最小宽度，也没有做水平居中（内容靠左显示）

### 2. 素材背景色 (src/lib/constants.ts)

- 当前颜色定义：
  - video: `#4A3728`（深棕色）
  - audio: `#E8A840`（金黄色）
  - image: `#F5E6C8`（米色）
  - text: `transparent`
- `OVERLAY_AUDIO_COLOR = "#F0A0B0"`（非主轨音频颜色）
- 需求中 subtitle 对应 item.type = "text"，将 text 颜色改为 #C9F0FF

### 3. 高分辨率适配 (thumbnail-extractor.ts, waveform-extractor.ts)

- **缩略图**：`thumbnail-extractor.ts` 中 canvas 尺寸直接设为 CSS 像素值（`width = thumbWidth, height = MAIN_TRACK_HEIGHT`），未乘以 devicePixelRatio，在 Retina 屏上模糊
- **波形图**：`WaveformCanvas.tsx` 中 `canvas.width = width; canvas.height = height` 同样未考虑 devicePixelRatio

### 4. 波形图圆角 (WaveformCanvas.tsx)

- 当前使用 `ctx.fillRect()` 绘制 bar，无圆角
- 需要改为 `ctx.roundRect()` 绘制，四角均为 2px 圆角（bar 宽度 2px 时圆角 = min(2, barWidth/2) = 1px，呈胶囊形）

## 技术实现方案

### 方案一：Demo 布局优化

- 在 `src/demo/App.tsx` 中给最外层容器添加 `min-w-[1400px]` 和水平居中样式
- 时间轴保持 800px，预览播放区域保持 600px

### 方案二：素材背景色更新

- 修改 `src/lib/constants.ts` 中 `ITEM_COLORS`：
  - video: `#4A3728` -> `#FF8702`
  - audio: `#E8A840` -> `#FFCEE1`
  - image: `#F5E6C8` -> `#B7E5D2`
  - text: `transparent` -> `#C9F0FF`（subtitle 对应 text 类型）
- 删除 `OVERLAY_AUDIO_COLOR` 常量
- 修改 `TrackItem.tsx`：移除非主轨音频的特殊颜色逻辑，统一使用 `ITEM_COLORS[item.type]`
- 波形图 bar 颜色暂不修改

### 方案三：Retina 屏高清适配

- 统一使用 `window.devicePixelRatio`（简称 dpr）
- **缩略图 canvas**（`thumbnail-extractor.ts`）：
  - canvas 内部分辨率设为 `width * dpr, height * dpr`
  - 通过 CSS 样式保持 canvas 显示尺寸不变
  - CanvasSink 的 width/height 需要传入 dpr 倍的值
- **波形图 canvas**（`WaveformCanvas.tsx`）：
  - canvas.width/height 设为 `width * dpr, height * dpr`
  - 通过 `canvas.style.width/height` 保持 CSS 尺寸不变
  - 绘制前 `ctx.scale(dpr, dpr)`，后续绘制逻辑坐标不变

### 方案四：波形图 bar 圆角

- 在 `WaveformCanvas.tsx` 中将 `ctx.fillRect()` 替换为 `ctx.beginPath()` + `ctx.roundRect()` + `ctx.fill()`
- 圆角半径 = `Math.min(2, barWidth / 2)`（2px bar 宽度下实际为 1px，胶囊形）
