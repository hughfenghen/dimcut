# 时间轴音频元素波形图

## 背景

- 基本已经按照 ./desgin.md 实现了 Item 的绘制与交互
- 已实现视频元素的缩略图提取 ./260503.feat.video-thumbnails.md

## 需求

- 给视频音轨、音频元素添加一个波形图
  - 如果视频元素存在音轨，在缩略图底部增加 20px 的高度绘制波形图
  - 新建 AudioTrackItem 组件，支持波形图，元素高度保持当前不变
- 使用 Mediabunny 获取音频数据，使用 canvas 绘制波形图
  - 期望 VideoTrackItem AudioTrackItem 能复用波形图组件
  - 波形图组件需支持 颜色、line 宽度、gap 宽度、最大高度、sampleRate 等基本设置
    - 波形图的内容是音频时域数据，可视化为 bar
  - 通过 Pixels per second 动态调节 sampleRate 参数，让波形图疏密适中
- 建议实现路径
  - 每行根据起始结束时间，通过 AudioBufferSink 获取对应的 buffer
  - 从 buffer 提取出二进制数据，通过 canvas 绘制
  - 参考 video thumbnails 实现懒加载与缓存策略
    - 加载可视窗口内音频元素的波形图，并缓存
    - Pixels per second 和 容器宽度变化，缓存失效, 更新波形图
  - 注意 buffer 包含一个时间区间的音频数据，AudioTrackItem 每行需要组合多个 buffer，并从 头尾 buffer 中截取片段
- 示例代码：

```ts
import { AudioBufferSink } from "mediabunny";

const sink = new AudioBufferSink(audioTrack);
const audioContext = new AudioContext();

for await (const { buffer, timestamp, duration } of sink.buffers()) {
  // 使用 Web Audio API 进行分析
  const analyser = audioContext.createAnalyser();
  const source = audioContext.createBufferSource();
  source.buffer = buffer;
  source.connect(analyser);

  // 获取时域数据（波形）
  const timeDomainData = new Uint8Array(analyser.frequencyBinCount);
  analyser.getByteTimeDomainData(timeDomainData);
}

// 可视化
analyser.fftSize = 2048;
const bufferLength = analyser.frequencyBinCount;
const dataArray = new Uint8Array(bufferLength);
analyser.getByteTimeDomainData(dataArray);

// draw an oscilloscope of the current audio source

function draw() {
  analyser.getByteTimeDomainData(dataArray);

  canvasCtx.fillStyle = "rgb(200 200 200)";
  canvasCtx.fillRect(0, 0, WIDTH, HEIGHT);

  canvasCtx.lineWidth = 2;
  canvasCtx.strokeStyle = "rgb(0 0 0)";

  canvasCtx.beginPath();

  const sliceWidth = (WIDTH * 1.0) / bufferLength;
  let x = 0;

  for (let i = 0; i < bufferLength; i++) {
    const v = dataArray[i] / 128.0;
    const y = (v * HEIGHT) / 2;

    if (i === 0) {
      canvasCtx.moveTo(x, y);
    } else {
      canvasCtx.lineTo(x, y);
    }

    x += sliceWidth;
  }

  canvasCtx.lineTo(canvas.width, canvas.height / 2);
  canvasCtx.stroke();
}
```

## 现状分析

### 现有架构

- **主轨渲染**: `TimelineRow.tsx` 中主轨区域根据 `item.type` 分发渲染：`video` 类型使用 `VideoTrackItem`，其他类型使用 `TrackItem`
- **VideoTrackItem**: 使用 canvas 绘制视频缩略图，高度固定 `MAIN_TRACK_HEIGHT = 40px`
- **TrackItem**: 纯色矩形，高度统一 `ROW_ITEM_HEIGHT = 32px`，audio 主轨颜色 `#E8A840`
- **ThumbnailExtractor**: 管理视频缩略图的提取和缓存，使用 mediabunny 的 `CanvasSink`
- **懒加载**: `VideoTrackItem` 通过 `IntersectionObserver` 实现可视区域懒加载
- **缓存策略**: `ThumbnailExtractor` 按 `step` 维护缓存 Map，`step` 变化时清除旧缓存

### Mediabunny AudioBufferSink API

- `AudioBufferSink(audioTrack: InputAudioTrack)` — 从音频轨道创建 sink
- `buffers(startTimestamp?, endTimestamp?)` — 异步迭代器，返回 `{ buffer: AudioBuffer, timestamp: number, duration: number }`
- `getBuffer(timestamp)` — 获取指定时间戳的单个 AudioBuffer
- `Input.getPrimaryAudioTrack()` — 获取主音频轨道（视频文件也可能包含音频轨道）

### AudioBuffer 数据提取

`AudioBuffer` 是 Web Audio API 的标准对象，可直接通过 `buffer.getChannelData(channel)` 获取 `Float32Array` 格式的 PCM 数据（值范围 -1.0 ~ 1.0），无需通过 `AudioContext` 和 `AnalyserNode`。这比 spec 示例代码中的方式更高效直接。

### 需要变更的点

- **高度变更**: 视频主轨若有音轨，高度需从 40px 增加到 60px（底部 20px 绘制波形图）；无音轨则保持 40px
- **新增组件**:
  - `WaveformCanvas` — 可复用的波形图 canvas 绘制组件（bar 白色，canvas 透明）
  - `AudioTrackItem` — 音频类型的渲染组件（含波形图），高度 `ROW_ITEM_HEIGHT = 32px`
- **新增工具类**: `WaveformExtractor` — 类似 `ThumbnailExtractor`，管理音频波形数据的提取与缓存
- **修改组件**: `VideoTrackItem` — 底部增加 20px 波形图区域（有音轨时）；`TimelineRow` — 增加对 audio 主轨和波形图的处理
- **高度计算**: `TimelineRow.totalContentHeight` 需要考虑视频有音轨时的额外 20px

## 技术实现方案

### 1. WaveformExtractor（波形数据提取器）

参考 `ThumbnailExtractor` 的设计模式，新建 `waveform-extractor.ts`：

- **初始化**: 接收 `File`，创建 mediabunny `Input`，获取 `getPrimaryAudioTrack()`，创建 `AudioBufferSink`
- **数据提取**: `extract(startTime, endTime, pps)` 返回 `Float32Array`（归一化后的振幅数据，每像素一个值）
  - 通过 `sink.buffers(startTime, endTime)` 获取时间范围内的所有 AudioBuffer
  - 从每个 buffer 通过 `buffer.getChannelData(0)` 获取 PCM 数据
  - 按 `samplesPerPixel = sampleRate / pps` 降采样：每 N 个采样取绝对值最大值（peak），生成与像素宽度对应的振幅数组
  - 多个 buffer 需要正确拼接，头尾 buffer 需根据时间裁剪
- **缓存策略**: 按 `pps` 维度缓存整段波形数据
  - `cache: Map<pps, Float32Array>`（全段波形数据）
  - pps 变化时旧缓存失效
  - 首次提取后，后续同 pps 下直接切片返回
- **sampleRate 计算**: `samplesPerPixel = audioSampleRate / pps`
  - 即每像素对应多少个原始采样点，取这些点中的 peak 值
- **hasAudioTrack()**: 返回是否存在音频轨道

### 2. WaveformCanvas（波形图 canvas 组件）

新建 `WaveformCanvas.tsx`，VideoTrackItem 和 AudioTrackItem 复用：

- **Props**:
  - `waveformData: Float32Array` — 振幅数据（0~1 归一化）
  - `width: number` — canvas 像素宽度
  - `height: number` — canvas 像素高度
  - `color?: string` — bar 颜色（默认白色 `#FFFFFF`）
  - `barWidth?: number` — 单个 bar 宽度（默认 2px）
  - `gap?: number` — bar 间距（默认 1px）
- **绘制逻辑**:
  - canvas 背景透明，由外层容器背景色区分场景
  - 计算每个 bar 对应的数据索引范围，取范围内 peak 值
  - 从 canvas 底部向上绘制 bar
  - bar 高度 = peakValue \* maxHeight

### 3. VideoTrackItem 改造

- 有音轨时：高度从 `MAIN_TRACK_HEIGHT(40px)` 增加到 `40 + 20 = 60px`（底部 20px 写死，不设常量）
- 无音轨时：高度保持 `MAIN_TRACK_HEIGHT(40px)` 不变
- 底部 20px 区域放置 `WaveformCanvas`（白色 bar + 视频底色背景）
- 缩略图 canvas 仍然绘制在上部 40px 区域
- 需要传入 `WaveformExtractor` 实例
- 波形数据的获取通过 `WaveformExtractor.extract(visibleStart, visibleEnd, pps)` 实现

### 4. AudioTrackItem（新组件）

新建 `AudioTrackItem.tsx`，用于主轨 audio 类型的渲染：

- 高度 `ROW_ITEM_HEIGHT = 32px`
- 底色 `ITEM_COLORS.audio (#E8A840)` + 白色 bar 波形图覆盖
- 内部放置 `WaveformCanvas`，高度 = 32px
- 懒加载策略与 `VideoTrackItem` 一致（`IntersectionObserver`）
- 需要 `WaveformExtractor` 实例

### 5. TimelineRow 改造

- 主轨区域条件渲染：`video` → `VideoTrackItem`（有音轨时含波形），`audio` → `AudioTrackItem`
- 主轨高度计算：若 video 且有音轨，`MAIN_TRACK_HEIGHT + 20`；若 audio，`ROW_ITEM_HEIGHT`；否则 `MAIN_TRACK_HEIGHT`
- 传入 `waveformExtractor` prop
- `totalContentHeight` 中的 main 部分使用动态主轨高度

### 6. Timeline 改造

- 创建 `WaveformExtractor` 实例（当主轨文件存在时，无论 video 或 audio）
- 将 `waveformExtractor` 传递给 `TimelineRow`
- 生命周期管理：`onCleanup` 时 dispose

### 7. 导出更新

- `index.ts` 导出 `AudioTrackItem`、`WaveformExtractor`

## bug

- [x] 轨道的 top 偏移异常，因为 VideoTrackItem 的高度可能是 40px 或 60px
  - 原因：overlay 层的 `top` 使用普通变量计算，非响应式，当 waveformExtractor 异步加载后 mainTrackHeight 变化时不会更新
  - 修复：将 `top` 改为响应式函数 `top()`，在 JSX 中调用以确保 Solid.js 追踪依赖
- [x] 波形图 canvas 中的 bar 应该沿垂直中分水平线镜像，垂直方向对称，垂直方向居中对齐
  - 修复：以 canvas 垂直中心为基线，bar 向上下两侧等量延伸
- [x] 优化 bar 的高度计算策略，当前 bar 的高度太矮了，垂直方向有大量的空白空间
  - 原因：peak 值为原始绝对值（0~1），大部分音频的 peak 远小于 1.0
  - 修复：对可见范围内所有 bar 的 peak 进行归一化（除以最大 peak），使最高 bar 填满可用高度
