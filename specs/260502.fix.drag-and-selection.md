# 修复无法拖拽和框选的问题

## 背景

- 基本已经按照 ./desgin.md 进行实现

## bug

- 非 MainTrackItem 的 Item 无法拖拽
  - 期望可以通过鼠标拖拽进行移动，改变 startTime 值
  - 期望可以对 Item 进行跨行移动
- 无法通过鼠标框选区域，没有绘制框选效果
  - 期望可以通过鼠标框选区域，选区支持跨行
  - 框选区域显示为蓝色半透明矩形，框选过程也需要实时更新
  - 存在框选区域显示浮动菜单，在框选区域上方，菜单 icon 横向排列
    - 暂时只有一个 delete icon, 点击添加删除区间
    - 新增 API 支持自定义菜单项
- 文字轨内容、左侧的时间都禁止选中效果，避免视觉干扰

- 删除区间（半透明红色）阻挡了下层 Item
  - 期望可以使用鼠标拖动下层的 Item 元素，即鼠标可穿透删除区间元素，但需要确保删除区间右脚右上角的 icon 可点击
- Item 无法上下方向跨行拖拽移动
  - 当前 Item 元素可以左右移动，期望同时支持拖拽元素快速上下跨多行移动

## 现状分析

### Bug 1：非 MainTrackItem 无法拖拽

**根因分析：**

`Timeline.tsx:handleItemDragStart` 中拖拽逻辑存在引用比较问题：

```ts
const onMove = (me: MouseEvent) => {
  const di = dragItem();
  // ...
  const newItems = props.data.items.map((it) =>
    it === di // ← 问题：di 是拖拽开始时保存的旧引用
      ? { ...it, startTime: newStart, endTime: newStart + duration }
      : it,
  );
  props.onChange?.({ ...props.data, items: newItems });
};
```

第一次 mousemove 触发 onChange 后，外部 `setData` 更新了 `data` signal，`props.data.items` 变成了新数组（含新对象引用）。后续的 mousemove 中 `dragItem()` 保存的还是旧引用，`it === di` 永远为 false，导致拖拽无效。

**跨行拖拽：** 当前拖拽逻辑仅通过 `dx / pps()` 计算时间偏移，`startTime` 改变后 Item 会自动被分配到新的行（`assignItemsToRows` 按时间范围分配），所以跨行在逻辑上已经支持。但首次 move 后就因引用比较失败而中断。

### Bug 2：框选无绘制效果

**根因分析：**

`Timeline.tsx` 中 `rangeStart`/`rangeEnd` signal 在拖拽过程中被更新，但没有在 JSX 中渲染任何可视化的框选矩形。框选效果需要：

1. 在拖拽过程中实时渲染蓝色半透明矩形
2. 框选矩形需要跨行显示（当前框选仅绑定到单行的 `rowStartTime`，不支持跨行）
3. 框选结束后显示浮动菜单（目前完全没有实现）

### Bug 3：文字轨和时间标签可选中

**根因分析：**

- `AsrTrack.tsx` 中的文字 `<span>` 没有 `user-select: none` 样式
- `TimelineRow.tsx` 中的时间标签 `<div>` 也没有 `user-select: none` 样式
- 在拖拽/框选过程中，浏览器默认会触发文字选中行为

## 技术实现方案

### 1. 修复拖拽：为 Item 新增 id 字段 + 基于 id 比较

- 在 `IItem` 接口新增 `id: string` 字段
- `handleItemDragStart` 中保存拖拽 item 的 `id`（而非引用），onMove 中通过 `it.id === dragItemId` 定位 item
- 不再使用 `dragItem` signal 存储引用，改为存储 `dragItemId: string | null`
- 同时保存拖拽开始时的 `startTime` 和 item 时长，用于计算新位置
- mock-data.ts 中的 items 需要补充 id 字段

### 2. 实现框选可视化与跨行支持

**框选状态管理（在 Timeline.tsx 中）：**

- 新增信号：`selectionStart`（全局时间）、`selectionEnd`（全局时间）、`isSelecting`（布尔值）、`hasSelection`（布尔值，表示框选完成后选区是否仍然有效，用于显示菜单）
- mousedown 在空白区域时，通过行信息和行内 X 计算全局时间作为 `selectionStart`
- mousemove 时根据鼠标位置计算当前全局时间更新 `selectionEnd`（支持跨行：通过 timeline 容器 ref 和行 DOM 元素计算当前行）
- mouseup 时如果选区有效（时间差 >= 0.1s），保留选区并显示浮动菜单；否则清除

**跨行鼠标位置→全局时间转换：**

- Timeline 组件保存容器 ref
- mousemove 时通过 `containerRef.querySelectorAll('[data-row-index]')` 获取各行 DOM 元素
- 根据鼠标 Y 坐标判断当前行，结合 X 坐标和 `pixelToTime` 计算全局时间
- 如果鼠标在最上方行之上，clamp 到第一行起始；在最下方行之下，clamp 到最后行结束

**框选矩形渲染：**

- 在 `TimelineRow` 新增 `selectionRange?: { start: number; end: number }` prop
- Row 内部判断选区与自身行的时间重叠，渲染蓝色半透明矩形 `rgba(59, 130, 246, 0.2)`
- 矩形覆盖行的全部垂直高度

**浮动菜单：**

- 在 Timeline 层渲染，定位在选区最后一行对应的矩形上方
- 默认包含 delete 按钮（SVG trash icon），点击后将选区添加到 deletedRanges（含重叠合并）并触发 onChange
- 通过 `selectionMenuItems` prop 支持自定义菜单项，每项包含 `icon: () => JSX.Element`、`label: string`、`onClick: (selection) => void`

**deletedRanges 重叠合并算法：**

- 新增 `mergeDeletedRanges(ranges: DeletedRange[]): DeletedRange[]` 工具函数
- 按 start 排序后，顺序合并重叠/相邻区间
- 添加新 deletedRange 时先 push 再调用 merge

### 3. 禁止文字选中

- 在 Timeline 根容器 `<div>` 添加 `select-none` class
- 所有子元素自动继承 user-select: none

### 4. API 变更

`types.ts` 变更：

```ts
// IItem 新增 id
interface IItem {
  id: string; // 新增
  type: string;
  startTime: number;
  endTime: number;
  zIndex: number;
}

// 新增 SelectionMenuItem
interface SelectionMenuItem {
  icon: () => JSX.Element;
  label: string;
  onClick: (selection: { start: number; end: number }) => void;
}

// TimelineProps 新增
interface TimelineProps {
  data: IChangeEventData;
  pixelsPerSecond?: number;
  onChange?: (data: IChangeEventData) => void;
  selectionMenuItems?: SelectionMenuItem[]; // 新增
}
```
