## 背景

- 当前已经实现视频导出 @specs/260505.feat.export-video.md

## 需求

- basic demo 需要支持添加片段功能
  - 在浮动菜单中添加一个 右箭头 icon
  - 添加 icon 将当前选中的时间片段添加到片段列表中
  - 片段列表元素添加到右侧，预览播放器下方 @src/demo/App.tsx L188-L197

  - 片段可以拖拽调整顺序，导出片段时需要重置帧时间戳

    - 按钮名称可以重命名，跟已有的导出按钮冲突，可能产生迷惑
- 导出片段
  - 与已有的导出功能相反
    - 已有导出剔除片段
    - 本导出保留指定片段，且时间顺序可以被调整
  - 从 @src/lib/video-exporter.ts 中提取出能复用的代码，实现导出功能
- 请根据上述内容，给"片段"起一个合适的容易理解的名字
  - 无片段时的空状态，提供良好的引导说明文案

## 技术实现方案

### 1. 命名方案

- 片段统一称为 **"片段"（Clip）**
- 已有导出按钮重命名为 **"导出裁剪"**（表示导出剔除删除区间后的结果）
- 新导出按钮命名为 **"导出片段"**
- 空状态引导文案：**"在时间轴上选择区间，点击 → 按钮添加片段"**

### 2. 数据结构

```ts
interface Clip {
  id: string; // 唯一标识，用于列表 key 和拖拽
  start: number; // 起始时间（秒）
  end: number; // 结束时间（秒）
}
```

- 在 `App.tsx` 中新增 `clips` signal：`createSignal<Clip[]>([])`
- id 使用 `crypto.randomUUID()`
- 允许重叠片段（同一时间区间可多次添加）

### 3. 浮动菜单添加 "添加片段" 按钮

- 通过 Timeline 的 `selectionMenuItems` prop 传入自定义菜单项
- icon 使用右箭头 SVG（chevron-right 风格）
- onClick 回调将 `{start, end}` 追加到 `clips` signal

### 4. 片段列表 UI（App.tsx 内实现）

- 位于 PreviewPlayer 下方，包裹在同一个右侧容器中
- 右侧容器设为 `sticky top-4`，`max-h-screen overflow-y-auto`
- 列表结构：
  - 标题区："片段" + "导出片段"按钮
  - 空状态：引导文案
  - 片段卡片列表：
    - 每个卡片显示序号、起始-结束时间（使用 `formatTime` from `time-utils.ts`，格式 `mm:ss.ss`）
    - 点击卡片 → `setCurrentTime(clip.start)`
    - 删除按钮（× 图标）
    - 拖拽手柄（拖拽排序）

### 5. 拖拽排序

- 使用原生 HTML5 拖拽 API（`draggable`、`dragstart`、`dragover`、`drop`）
- 实现方式：
  - 卡片设置 `draggable={true}`
  - `dragstart` 记录拖拽源索引
  - `dragover` 计算目标位置，显示插入指示器
  - `drop` 重新排列 `clips` 数组

### 6. 重构 video-exporter.ts

- 从 `exportVideo` 中提取公共函数：

```ts
// 公共：创建 mediabunny input/output 并启动
async function createExportPipeline(file: File): Promise<{
  input: Input;
  videoTrack: ...;
  audioTrack: ...;
  videoSource: VideoSampleSource;
  audioSource: AudioSampleSource;
  output: Output;
  writableStream: FileSystemWritableFileStream;
}>
```

- `exportVideo` 改为调用 `createExportPipeline`，帧遍历逻辑保留在原函数中
- 新增 `exportClips` 函数：

```ts
export async function exportClips(
  file: File,
  clips: Clip[], // 已排序的片段列表
): Promise<void>;
```

- `exportClips` 的时间戳调整策略：
  - 按 clips 数组顺序拼接，每个片段的帧时间戳需要连续
  - 维护 `accumulatedDuration`，第 N 个片段的帧时间戳 = `frame.timestamp - clip.start + accumulatedDuration`
  - 每个片段处理完后 `accumulatedDuration += clip.end - clip.start`

### 7. 时间格式化

- 复用 `src/lib/time-utils.ts` 中已有的 `formatTime(seconds)` 函数，输出 `mm:ss.ss` 格式

## bugs

- [x] 右侧预览播放的 video 元素可能宽度溢出
  - PreviewPlayer 外层 div 从 `w-[600px] sticky top-4` 改为 `w-full`，移除冗余 sticky（父容器已有）

- [x] 导出片段报错：`Error: Timestamps must be non-negative (got -0.007333333333333414s)`
  - 根因：`exportClips` 中 `frame.timestamp - clip.start` 计算时，媒体解码器返回的帧时间戳可能略早于请求的 `clip.start`（解码粒度/关键帧对齐问题），差值为负数
  - 修复：在 `video-exporter.ts` 的 `exportClips` 函数中，对视频帧（L184）和音频帧（L200）的 `setTimestamp` 参数添加 `Math.max(0, ...)` 防护
  - 验收点：导出片段不再报负时间戳错误

- [x] `await pipeline.writableStream.close()` (video-exporter.ts:124) 视频导出成功，但控制台打印错误
  - `Uncaught (in promise) TypeError: Failed to execute 'close' on 'WritableStream': Cannot close a locked stream`
  - 根因：`output.finalize()` 内部通过 `StreamTarget` 已关闭底层 `WritableStream`，再次调用 `writableStream.close()` 时流已被锁定
  - 修复：移除 `finalizePipeline` 中 `await pipeline.writableStream.close()` 行，同时从 `ExportPipeline` 接口中移除 `writableStream` 字段（不再需要持有引用）
  - 验收点：导出视频/片段后控制台无 `Cannot close a locked stream` 错误
