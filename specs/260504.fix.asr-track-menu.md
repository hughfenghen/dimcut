## 现状分析

### 数据流现状

| 用户操作 | 事件路径 | 状态影响 |
|---------|---------|---------|
| 媒体轨道 mousedown | TimelineRow.handleMouseDown → Timeline.handleRangeSelectStart → mousemove/mouseup | selectionStart/End, isSelecting, hasSelection |
| ASR 轨道 mousedown | TimelineRow.handleMouseDown → return（被跳过） | 无 Solid 状态变化，仅浏览器原生文本选择 |
| 点击 track-item | TimelineRow.handleMouseDown → return → handleItemDragStart | dragItemId, hasSelection=false |

### Bug 1：媒体轨道选区时菜单位置错误

**根因**：`menuPosition`（Timeline.tsx:324-370）用 `rowRect.top` 计算菜单 Y 坐标。`rowRect` 是整个行元素（含 ASR + 媒体区域）的边界矩形，`top` 指向行的最顶部（即 ASR 轨道上方）。而 selection overlay 仅覆盖 `nonAsrTop()` 以下的媒体区域（TimelineRow.tsx:278-293）。导致菜单定位在 ASR 轨道上方而非媒体选区上方。

### Bug 2：ASR 文字选择时菜单不可见

**根因**：功能缺失。TimelineRow.handleMouseDown（TimelineRow.tsx:126）检测到 `data-asr-track` 后直接 return，不触发 `onRangeSelectStart`，因此 `selectionRange`/`hasSelection` 永远不会被设置。没有任何代码将浏览器原生文本选择（基于 `data-asr-word-start/end` 属性）转换为 `selectionRange` 时间区间。

### Bug 3：选区不互斥

**根因**：两套选择系统完全独立。媒体选区由 `selectionStart/End/hasSelection` Solid 信号管理；ASR 文本选区使用浏览器原生 `window.getSelection()`。没有同步机制在一种选区激活时清除另一种。

## 技术实现方案

### 方案概述

引入 `selectionSource: 'media' | 'asr'` 信号，统一管理两种选区的互斥与菜单定位。

### Bug 1 修复：菜单定位修正

在 `menuPosition`（Timeline.tsx:324-370）中，当 `selectionSource === 'media'` 时，通过 DOM 查询 `[data-asr-track]` 元素获取 ASR 高度偏移量，将 Y 坐标加上该偏移量。

- 从 `firstRowEl` 中查询 `[data-asr-track]` 元素，获取其 `offsetHeight`
- `topY = rowRect.top - containerRect.top + asrTrackOffset`
- 使用 DOM 查询方式，不增加组件间耦合

### Bug 2 修复：ASR 文本选区转时间区间

1. 在 Timeline 组件中通过 `createEffect` + `document.addEventListener('selectionchange', ...)` 监听浏览器选区变化
2. 当检测到 `window.getSelection()` 的 range 起止元素都带有 `data-asr-word` 属性时：
   - 读取起止元素的 `data-asr-word-start` / `data-asr-word-end` 属性获取时间区间
   - 设置 `selectionSource` 为 `'asr'`
   - 设置 `selectionStart`/`selectionEnd` 为 ASR 时间区间
   - 设置 `hasSelection` 为 true
3. ASR 选区清除时（selection 为空或不在 ASR 元素内），清除 `hasSelection` 和 `selectionSource`
4. 删除按钮复用现有 `handleDeleteSelection`，ASR 选区产生的时间区间可直接映射为 DeletedRange
5. 视觉反馈仅使用浏览器原生文本高亮，不额外添加蓝色 overlay

### Bug 3 修复：选区互斥

1. 在 `handleRangeSelectStart`（Timeline.tsx:255）开始时调用 `window.getSelection()?.removeAllRanges()` 清除浏览器文本选择，设置 `selectionSource` 为 `'media'`
2. 在 ASR `selectionchange` 回调检测到 ASR 选区时，先设置 `selectionSource` 为 `'asr'`，清除媒体选区状态
3. 通过 `selectionSource` 标记当前选区来源，确保状态一致性
