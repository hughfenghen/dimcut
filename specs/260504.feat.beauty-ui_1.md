## 技术实现方案

### T1: 时间格式保留两位小数

- 修改 `formatTime` 函数，输出格式改为 `MM:SS.xx`
- 使用 `toFixed(2)` 处理秒的小数部分
- 影响范围：`src/lib/time-utils.ts`

### T2: Pixels per second 控件美化

- 移除 `<label>` 文字描述
- 使用 SVG 圆圈包裹 `+` / `-` 图标，置于 slider 前后
  - 点击 `+`：pps 增大（乘以 1.2，clamp 到 max=600）
  - 点击 `-`：pps 减小（乘以 0.8，clamp 到 min=5）
- slider 自定义样式：细线轨道、黑色滑块、黑色轨道
  - 使用 Tailwind `accent-black` 或自定义 CSS
- 影响范围：`src/demo/App.tsx`

### T3: Timeline 支持显示/隐藏轨道

- `TimelineProps` 新增两个可选 prop：
  - `showAsrTrack?: boolean`（默认 true）
  - `showMaterialTracks?: boolean`（默认 true）
- `Timeline` 内部将这两个 prop 传递给 `TimelineRow`
- `TimelineRow` 中：
  - 当 `showAsrTrack === false` 时跳过 `AsrTrack` 渲染
  - 当 `showMaterialTracks === false` 时跳过 overlay layers 渲染
- Demo App 新增两个黑色 checkbox 控制开关
- 影响范围：`src/lib/types.ts`、`src/lib/Timeline.tsx`、`src/lib/TimelineRow.tsx`、`src/demo/App.tsx`

### T4: 时间指示器仅覆盖素材轨道区

- 在 `Timeline.tsx` 的 playhead 渲染逻辑中：
  - `top` 从原来的 `el.offsetTop`（整行顶部）改为 `el.offsetTop + mainTrackTop`
  - `height` 从 `el.offsetHeight` 改为 `el.offsetHeight - mainTrackTop`
- 需要从 `TimelineRow` 获取 `mainTrackTop` 值，可通过 `data-main-track-top` 属性暴露
- 或者直接从 DOM 计算：找到 `[data-asr-track]` 元素的高度，playhead 从 ASR 底部开始
- 影响范围：`src/lib/Timeline.tsx`、可能需调整 `src/lib/TimelineRow.tsx`

### T5: ASR 文字轨保持段落分段

- 新增 `getAsrSegmentsForRow` 函数，返回按段落分组的结果：`AsrSegment[]`（仅包含当前 row 范围内的 words）
- 修改 `AsrTrack` 组件：
  - 外层 `<For>` 遍历 segments
  - 每个 segment 内部 `<For>` 遍历 words
  - segments 之间添加间距或分隔线（如 `margin-top: 4px`）
- 影响范围：`src/lib/asr-utils.ts`、`src/lib/AsrTrack.tsx`

## Bug 现状分析

### B1: 素材轨道开关仅控制 overlay layers，不含主轨道

- `showMaterialTracks` prop 传入 `TimelineRow`，在 `TimelineRow.tsx:46-47` 中仅控制 `effectiveLayers()`（overlay layers）
- 主轨道（`mainTrackSlice`）在 `TimelineRow.tsx:174-223` 始终渲染，不受 `showMaterialTracks` 影响
- 用户期望该开关控制"所有非 ASR 轨道"，即主轨道 + overlay layers 均受控
- Demo App 中 checkbox 标签为"素材轨道"，语义不够准确

### B2: pps 变更时 playhead 的 top 位置不更新

- `Timeline.tsx:497-510` 中 playhead 的 `top()` 和 `height()` 依赖 DOM 属性 `el.offsetTop` 和 `el.offsetHeight`
- 这些 DOM 属性通过 `containerRef.querySelector` 读取，非 SolidJS 响应式依赖
- 当 `pps()` 变化时，行高和行位置会变化（因为 ASR/overlay 高度可能不同），但 `top()`/`height()` 不会重新计算
- 根本原因：`top()` 和 `height()` 函数内部没有任何响应式信号依赖，SolidJS 无法追踪其变化

### B3: ASR 文字选中时菜单位置计算错误

- `Timeline.tsx:380-435` 中 `menuPosition` 的计算逻辑：
  - 对 `selectionSource === "media"` 的情况，会偏移 `topY` 到 ASR 底部（L425-432）
  - 对 `selectionSource === "asr"` 的情况，不做任何偏移，直接使用 `rowRect.top - containerRect.top` 作为 `topY`
- 这导致 ASR 选中时菜单定位在行的最顶部（ASR 轨道上方），而非选中文字的正上方
- 用户期望：菜单应显示在 `window.getSelection().getRangeAt(0)` 的 `startContainer` 对应 DOM 元素的上方

## Bug 技术方案

### B1: 素材轨道开关含主轨道 + 重命名

- 将 `showMaterialTracks` 重命名为 `showMediaTracks`（更准确表达"所有媒体轨道"）
- `TimelineRow.tsx` 中增加 `effectiveMainTrack()` 计算函数：
  - `showMediaTracks === false` 时返回 `undefined`（不渲染主轨道）
  - 否则返回 `props.mainTrackSlice`
- `totalContentHeight` / `mainTrackTop` / `overlayBaseTop` 等高度计算函数需同步适配
- Demo App checkbox 标签改为"媒体轨道"
- 影响范围：`src/lib/types.ts`、`src/lib/Timeline.tsx`、`src/lib/TimelineRow.tsx`、`src/demo/App.tsx`

### B2: playhead 位置响应 pps 变更

- 在 `top()` 和 `height()` 函数中引入 `pps()` 作为响应式依赖，使 SolidJS 在 pps 变化时重新计算
- 方案：在 `top()` 函数体开头读取 `pps()`（即使不使用其值），建立响应式追踪
- 同理对 `height()` 做相同处理
- 影响范围：`src/lib/Timeline.tsx`

### B3: ASR 选中菜单定位到选中文字上方

- 修改 `menuPosition` 计算逻辑，当 `selectionSource() === "asr"` 时：
  - 获取 `window.getSelection()?.getRangeAt(0)` 的 `getBoundingClientRect()`
  - 取 `rangeRect.left + rangeRect.width / 2` 作为 `x`（水平居中于选区）
  - 取 `rangeRect.top - containerRect.top` 作为 `y`（选区上方）
- 移除对 `selectionSource === "media"` 的特殊偏移逻辑（之前的 ASR 高度偏移在 ASR 选中也应同样处理）
- 影响范围：`src/lib/Timeline.tsx`
