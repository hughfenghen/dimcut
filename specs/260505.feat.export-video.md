  - 在demo 添加导出按钮，靠右 @src/demo/App.tsx L76-L134
- 导出实现逻辑
  - 根据视频时长和 deletedRanges，提取出有效/未被删除的片段
  - 将片段合并成一个新的视频文件
    - 音视频帧数据的时间戳调整，需要减去帧之前被删除片段的时间
  - 参考示例代码

```ts
import {
  Input,
  Output,
  VideoSampleSink,
  AudioSampleSink,
  VideoSampleSource,
  AudioSampleSource,
  Mp4OutputFormat,
  BufferTarget,
  BlobSource,
  ALL_FORMATS,
} from "mediabunny";

async function reencodeSegment(videoFile, startTime, endTime) {
  // 点击导出请求文件保存权限
  const fileHandle = await window.showSaveFilePicker({
    suggestedName: "video.mp4",
    types: [
      {
        description: "MP4 Video",
        accept: { "video/mp4": [".mp4"] },
      },
    ],
  });

  // 创建可写流
  const writableStream = await fileHandle.createWritable();

  // 创建输出，使用 StreamTarget
  const output = new Output({
    format: new Mp4OutputFormat(),
    target: new StreamTarget(writableStream, {
      chunked: true, // 批量写入以提高性能
    }),
  });

  // 1. 创建输入和 sinks
  const input = new Input({
    formats: ALL_FORMATS,
    source: new BlobSource(videoFile),
  });

  const videoTrack = await input.getPrimaryVideoTrack();
  const audioTrack = await input.getPrimaryAudioTrack();

  // 3. 创建视频和音频 sources
  const videoSource = new VideoSampleSource({
    codec: "avc",
    bitrate: 20e6, // 5 Mbps
  });

  const audioSource = new AudioSampleSource({
    codec: "aac",
    bitrate: 128e3, // 128 kbps
  });

  output.addVideoTrack(videoSource);
  output.addAudioTrack(audioSource);

  await output.start();

  // 4. 提取并重新编码视频帧
  if (videoTrack) {
    const videoSink = new VideoSampleSink(videoTrack);

    for await (const frame of videoSink.samples(startTime, endTime)) {
      // 调整时间戳，从 0 开始
      const adjustedFrame = new VideoSample(frame.frame, {
        timestamp: frame.timestamp - startTime,
        duration: frame.duration,
      });

      await videoSource.add(adjustedFrame);
      frame.close(); // 释放原始帧
    }
  }

  // 5. 提取并重新编码音频数据
  if (audioTrack) {
    const audioSink = new AudioSampleSink(audioTrack);

    for await (const audioData of audioSink.samples(startTime, endTime)) {
      // 调整时间戳，从 0 开始
      const adjustedAudio = new AudioSample(audioData.data, {
        timestamp: audioData.timestamp - startTime,
        duration: audioData.duration,
        sampleRate: audioData.sampleRate,
        numberOfChannels: audioData.numberOfChannels,
      });

      await audioSource.add(adjustedAudio);
      audioData.close(); // 释放原始音频数据
    }
  }

  // 6. 完成编码
  videoSource.close();
  audioSource.close();
  await output.finalize();

  return output.target.buffer;
}
```

## 现状分析

- 项目使用 `mediabunny` 库（^1.43.1）处理媒体，已有 `ThumbnailExtractor` 和 `WaveformExtractor` 作为参考
- `DeletedRange` 类型定义在 `src/lib/types.ts:55-58`，结构为 `{ start: number; end: number }`
- Demo 中 `deletedRanges` 状态由 Timeline 的 `onChange` 回调更新，存储在 `App.tsx` 的 signal 中
- mediabunny API 关键能力：
  - `Input` + `BlobSource` 打开文件
  - `VideoSampleSink.samples(start, end)` 按时间范围迭代解码视频帧
  - `AudioSampleSink.samples(start, end)` 按时间范围迭代解码音频样本
  - `VideoSampleSource` / `AudioSampleSource` 接收原始帧并编码
  - `Output` + `Mp4OutputFormat` + `StreamTarget` 流式写出 MP4（避免 OOM）
  - `VideoSample` 支持 `setTimestamp()` 方法直接修改时间戳（无需重建对象）
  - `AudioSample` 同样支持 `setTimestamp()` 方法
- 示例代码中用 `new VideoSample(frame.frame, ...)` 构造方式与实际 API 不符；实际 `VideoSample` 构造器接收 `VideoFrame | CanvasImageSource | AllowSharedBufferSource`，不接收 `frame.frame`。应直接用 `frame.setTimestamp(newTs)` 修改时间戳后传入 source

## 技术实现方案

### 1. 导出核心逻辑（新建 `src/lib/video-exporter.ts`）

- 输入：`file: File`，`deletedRanges: DeletedRange[]`，`duration: number`
- 计算有效片段：从 `[0, duration]` 中减去所有 `deletedRanges`，得到有效时间区间列表
- 对每个有效片段，使用 `VideoSampleSink.samples(start, end)` 和 `AudioSampleSink.samples(start, end)` 提取帧
- 时间戳调整策略：
  - 维护一个 `timeOffset` 累计值，表示当前帧之前所有被删除区间的总时长
  - 每个帧的新时间戳 = `原始时间戳 - timeOffset`
  - 直接使用 `sample.setTimestamp(newTs)` 修改，无需重建对象
- 按顺序将所有有效片段的视频帧和音频帧分别写入对应的 Source
- 使用 `StreamTarget` 流式写入文件，避免长视频 OOM

### 2. 输出方式

- 使用 `showSaveFilePicker` 获取文件句柄，创建 `FileSystemWritableFileStream`
- 通过 `StreamTarget(writableStream, { chunked: true })` 流式写入磁盘
- 数据不在内存中缓存，适合长视频导出

### 3. Demo UI 改动（`src/demo/App.tsx`）

- 在控制栏（L76-L134 的 flex 容器）右侧添加"导出"按钮
- 按钮点击时调用导出函数，传入当前视频文件、deletedRanges 和 duration
- 导出中显示加载状态，完成后自动下载

### 4. 视频/音频帧处理流程

```
有效片段 = computeValidSegments(duration, deletedRanges)
timeOffset = 0

for each segment in 有效片段:
  timeOffset = 计算该片段 start 之前所有 deletedRanges 的总时长
  for each videoFrame in videoSink.samples(segment.start, segment.end):
    videoFrame.setTimestamp(videoFrame.timestamp - timeOffset)
    await videoSource.add(videoFrame)
    videoFrame.close()
  // 音频同理
```

### 5. 编码配置

- 视频：codec `"avc"`, bitrate `5e6`（5 Mbps）
- 音频：codec `"aac"`, bitrate `128e3`（128 kbps）
