# 修复缩略图错误刷新

## 背景

已经实现了视频缩略图功能，./260503.feat.video-thumbnails.md

## bug

- deletedRanges 触发缩略图刷新的原因是 触发了 onChange ， 从而导致 currentData 值产生变化
  - 让整个 TimeLine 组件重新渲染了 @src/demo/App.tsx L86-L93，所以触发缩略图刷新
  - 修复方案：重构 TimeLine 组件，data 仅作为初始数据， onChange 事件提供最新状态，但组件后续状态不受父组件控制
    - data 改名为 initData

## 现状分析

- App.tsx 中 `currentData()` 是一个响应式计算，onChange 调用 setData 后值变化
- SolidJS 中 `{currentData() && <Timeline data={currentData()!} .../>}` 在信号变化时会重新执行表达式
- Timeline 内部所有 createMemo/createEffect 都直接引用 `props.data`，当 props.data 对象引用变化时全部重新计算
- 缩略图 extractor effect 监听 `props.data.mainTrackConf.item`，虽然理论上引用不变，但组件可能被整体重建导致 effect 重新执行

## 技术实现方案

- Timeline 组件内部创建 `items` 和 `deletedRanges` 信号，从 `props.initData` 初始化
- 所有内部操作（拖拽、删除区间）修改内部信号，同时调用 `props.onChange` 通知外部
- `mainTrackConf` 保持从 props 读取（不会变化）
- App.tsx 简化：不再需要 `data` signal 和 `currentData` 计算，直接传初始数据即可
- 缩略图 extractor effect 仅监听 `props.initData.mainTrackConf.item`（不变则不触发）

- 实施时将 Timeline 入参从 `data` 重命名为 `initData`，强调该值仅用于初始化内部状态
- `onChange` 作为状态回传通道，父组件不再反向驱动 Timeline 内部状态
