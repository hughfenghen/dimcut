## 现状分析

- 组件位置：`src/landing/TimelineIllustration.tsx`，使用 SolidJS + Tailwind 实现。
- 当前结构：
  - 顶部：浏览器风窗口栏（红/黄/绿小圆点 + `illust.title` 文本）。
  - 第一块：单个 ASR 文本卡片（`illust.asrText1/Highlight/Strike/Text2`），左侧有 `ml-7` 缩进。
  - 下方 3 行轨道（非需求要求的 4 行结构）：
    - row 01：视频块（slate 灰）+ 占位空白
    - row 02：空白 + 音频块（emerald 绿，含 `Wave` 波形）+ 空白 + 图片块（amber 黄）+ 浮层选区
    - row 03：空白 + 视频块（slate 灰）+ 空白
  - 每行第一列：硬编码序号 `01/02/03`，宽 `w-5`。
  - 顶层叠加：`top-12 bottom-3` 的播放头竖线 + 小圆点。
- 配色：当前轨道块使用 `slate/emerald/amber` 系，未使用 `src/lib/constants.ts` 中的 `ITEM_COLORS`。
- 波形 `Wave`：基于线性同余产生 0.25~1.0 的随机高度，柱子较细密。
- i18n：`illust.*` 当前仅 1 段 ASR 文案（zh/en 均如此），无第二段文字轨内容。
- 参考 SVG：对称音频图标（9 根圆角矩形，中间最高、两侧渐低），语义为"图标点缀"。
- `src/lib/constants.ts` 中有 `DELETED_RANGE_COLOR = rgba(255, 100, 100, 0.3)`，可用于"被删除片段"视觉。

## 技术实现方案

### 1. 整体结构重构（4 行：文字 → 素材 → 文字 → 素材）

- 移除当前位于轨道上方的"独立 ASR 卡片"区块；改为把文字轨纳入 4 行结构内，每行包含统一的 `[时间列 | 内容列]` 两栏。
- 4 行渲染顺序：
  1. text-row-A：文字轨内容（围绕"文字优先 / 二维折叠时间轴"）。
  2. media-row-A：上方文字对应的媒体片段（视频 + 音频 + 图片块的组合）。
  3. text-row-B：文字轨内容（围绕"文字编辑 = 视频编辑"，含"被删除文字"语义）。
  4. media-row-B：第二组媒体片段，含一段"被删除片段"红色样式呼应文字轨 B。
- 文字轨与素材轨**视觉成对**：通过等宽时间列让文字行的起始时间和下方素材行的起点严格对齐。

### 2. 时间列设计

- 第一列宽度统一为 `w-12`（兼顾 `00:00` 5 字符宽 + tabular-nums + 间距）。
- 时间标签仅在文字轨行显示：`text-row-A → 00:00`、`text-row-B → 00:12`；media 行的时间列保持空白。
- 文字与时间基线对齐，使用 `text-[--color-fg-faint]` + `tabular-nums`。

### 3. 媒体块配色（接入 ITEM_COLORS，柔化处理）

- 直接引用 `ITEM_COLORS` 常量（`#FF8702 / #FFCEE1 / #B7E5D2`），通过 `color-mix` / 透明度做柔化以贴合落地页极简风格：
  - 视频块：背景 `color-mix(in srgb, #FF8702 14%, white)`，边框 `color-mix(in srgb, #FF8702 35%, white)`，缩略帧条/分隔色用 `color-mix(in srgb, #FF8702 55%, white)`。
  - 音频块：背景 `color-mix(in srgb, #FFCEE1 55%, white)`，边框 `color-mix(in srgb, #FFCEE1 80%, #d96fa3)`，波形条颜色 `color-mix(in srgb, #d96fa3 65%, white)`。
  - 图片块：背景 `color-mix(in srgb, #B7E5D2 50%, white)`，边框 `color-mix(in srgb, #B7E5D2 70%, #4ea58a)`。
  - "被删除片段"块：底色用 `DELETED_RANGE_COLOR (rgba(255,100,100,0.3))`，叠加斜纹（`repeating-linear-gradient`）+ 边框虚线，呼应 text-row-B 的 `line-through` 文案。
- 完全移除媒体块内的 `illust.video / illust.audio / illust.img` 文字 label。
- i18n 中的 `illust.video / illust.audio / illust.img` 及 `illust.asrText1 / asrHighlight / asrStrike / asrText2` 一并清理（不保留）。

### 4. 音频波形 与 视频缩略帧条

- 音频块：重写 `Wave` 为对称凸起分布，9 根圆角矩形，高度比例固定为 `[0.38, 0.44, 0.19, 0.44, 1.0, 0.44, 0.25, 0.44, 0.13]`（参考 SVG 推算）。bar 宽更粗（`min-width: 3px`），圆角 `rounded-[2px]`，整体居中铺满。
- 视频块：改为"缩略帧条"装饰——一行等高、等宽的小矩形（约 5~6 帧），帧间留细缝（`gap-[2px]`），底色为视频块底色的更深 tone（`color-mix` 加深 10~15%），不使用任何图像。整体保持低饱和、极简风格，避免突兀。
- 两者作为独立组件 `WaveBars`（音频）与 `FrameStrip`（视频）共存于组件文件内部。

### 5. 文字轨文案（i18n，与素材轨显式呼应）

- 拆分为两段，删除旧 4 个 ASR key：
  - `illust.textTrackA`（围绕"二维折叠 / 文字优先"，呼应素材轨 A 的多素材横向排布）：
    - zh：把一维时间轴折叠为多行，{strike: 来回拖动波形} 改成 {highlight: 像读剧本一样阅读}。
    - en：Fold the 1D timeline into rows — {strike: scrubbing the waveform} becomes {highlight: reading like a script}.
  - `illust.textTrackB`（围绕"文字编辑 = 视频编辑"，呼应素材轨 B 的"被删除片段"红色样式）：
    - zh：{strike: 删除这段文字} 即 {highlight: 删除对应画面}，重排文字等于重排镜头。
    - en：{strike: Delete these words} → {highlight: delete the matching shots}. Reorder text, reorder cuts.
- i18n key 拆分方式：每段保留 `pre / strike / mid / highlight / post` 5 个 key 或者用更紧凑的 3-段切分（实现时以 5-段方案为准，便于复用现有的 highlight/line-through 渲染模式）：
  - `illust.textA.pre / textA.strike / textA.mid / textA.highlight / textA.post`
  - `illust.textB.pre / textB.strike / textB.mid / textB.highlight / textB.post`

### 6. 浮层装饰（加强）

- **选区浮层**：从文字轨 B 中的高亮文字位置出发，贯穿向下到 media-row-B 的"被删除片段"位置，使用 `--color-accent` 的 `border-x` + 半透明底色 + 顶部细标签（如 `selection`）形成"由文字定位、跨多行素材"的明确视觉链路。
- **播放头**：保留竖向播放头线 + 顶部小圆点，位置改为相对"时间列右侧"的百分比计算（如 `calc(48px + 24%)`），覆盖 4 行高度。

### 7. 保留要素

- 顶部窗口栏（红/黄/绿圆点 + `illust.title`）保留。
- 整体内边距、字号、`font-serif` 与 `text-[--color-fg-*]` 颜色变量沿用。
