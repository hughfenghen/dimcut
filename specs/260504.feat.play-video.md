# 预览播放素材，并关联时间轴组件

## 背景

- 当前已经基本实现了时间轴组件的交互 @src/lib/Timeline.tsx @src/lib/TimelineRow.tsx
- 包括音视频元素的缩略图和波形图 @src/lib/VideoTrackItem.tsx @src/lib/WaveformCanvas.tsx

## 需求

- 实现一个预览播放组件，接收 Timeline 组件一样的数据结构
  - 可预览播放停对应的数据结构
  - 暂时只渲染主轨的音视频元素，忽略其他素材
- 在 Timeline 上绘制时间指示器（红色竖线），与预览播放组件双向关联
  - 指示器：播放/暂停状态、当前播放的时间位置，与预览播放组件同步
  - 预览播放组件：支持 Timeline seek 时间指示器，跳到指定时间播放

## 流程状态

- 当前阶段：execute
- 最近动作：修复 bug（canvas 节点点击 seek）
- 更新时间：2026-05-04

## 现状分析

### 已有能力

1. **数据结构完备**：`IChangeEventData` 包含 `mainTrackConf`（主轨配置，含 `IVideoItem | IAudioItem` 和可选 `AsrData`）以及 `items`（覆盖层素材）
2. **媒体解码基础**：`ThumbnailExtractor` 和 `WaveformExtractor` 使用 `mediabunny` 库进行 WebAssembly 解码，但仅用于提取缩略图/波形，不用于播放
3. **Timeline 布局**：按行（row）+ 子行（subRow）组织，支持缩放（pixelsPerSecond）和容器自适应
4. **Demo 已加载视频 Blob**：从 `/example-video.mp4` 获取 Blob 并通过 HTML5 Video API 获取时长

### 缺失能力

- 无视频/音频播放器组件
- 无播放头（Playhead）/ 时间指示器组件
- 无播放状态管理（播放/暂停/seek）
- Timeline 组件目前不接受外部时间位置输入

### 技术栈

- Solid.js（响应式框架）、TypeScript、Vite、Tailwind CSS
- `mediabunny`（FFmpeg WASM 解码，仅用于提取）

## 技术实现方案

### 整体架构

采用"状态提升"模式：在 Demo App 层维护播放状态（currentTime、isPlaying），通过 props 分别传入 Preview 组件和 Timeline 组件，实现双向同步。

```
App (左右布局：左 Timeline 800px，右 PreviewPlayer 600px sticky)
├── Timeline (接收 initData, currentTime, onSeek)
├── PreviewPlayer (接收 mainTrackConf, currentTime, isPlaying, onSeek, onPlayPause)
```

### 1. PreviewPlayer 预览播放组件

- **位置**：`src/lib/PreviewPlayer.tsx`
- **职责**：播放主轨的音视频文件，提供播放/暂停/seek 控制
- **实现方式**：使用原生 HTML5 `<video>` 元素（主轨 file 为 Blob，通过 `URL.createObjectURL` 生成 src）
- **Props**：
  - `mainTrackConf: IMainTrackConf` — 主轨配置
  - `currentTime: number` — 当前播放时间（秒）
  - `isPlaying: boolean` — 播放状态
  - `onTimeUpdate: (time: number) => void` — 播放时持续上报当前时间
  - `onPlayPause: (playing: boolean) => void` — 播放/暂停状态变更回调
  - `onSeek: (time: number) => void` — seek 回调
- **视频尺寸**：固定宽度 600px，高度根据视频宽高比自适应
- **UI**：视频画面 + 底部播放/暂停按钮 + 时间显示；纯音频时仅显示播放控制（无画面）
- **同步逻辑**：
  - 外部 currentTime 变化时（来自 Timeline seek），调用 `video.currentTime = newTime`
  - 播放时通过 `requestAnimationFrame` 持续上报 `video.currentTime`

### 2. Timeline 时间指示器（Playhead）

- **位置**：在 `Timeline.tsx` 内部渲染，作为绝对定位的红色竖线
- **Props 扩展**：Timeline 新增 `currentTime?: number` 和 `onSeek?: (time: number) => void`
- **渲染逻辑**：
  - 根据 `currentTime` 和 `pixelsPerSecond` 计算指示器在当前可视区域的水平位置
  - 指示器跨越所有行，使用绝对定位 + `pointer-events: none`
  - 若 currentTime 对应位置不在当前可视行内，指示器不显示
- **Seek 交互**：
  - 若用户 mousedown 后未触发拖拽（未移动或移动距离极小），mouseup 时计算点击位置对应的时间，调用 `onSeek`
  - 与已有拖拽/选区交互兼容：拖拽一旦开始则不触发 seek

### 3. 播放状态管理（App 层）

- **状态**：`currentTime: number`、`isPlaying: boolean`
- **流向**：
  - PreviewPlayer `onTimeUpdate` → 更新 App `currentTime` → Timeline 指示器同步
  - Timeline `onSeek` → 更新 App `currentTime` → PreviewPlayer seek
  - PreviewPlayer `onPlayPause` → 更新 App `isPlaying`

### 4. Demo 集成

- 在 `src/demo/App.tsx` 中引入 PreviewPlayer
- 布局：左右结构，左边 Timeline（宽度 800px），右边 PreviewPlayer（宽度 600px，sticky 定位）
- 播放状态由 App 的 Solid.js signal 管理
