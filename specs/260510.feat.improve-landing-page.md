## 技术实现方案

### 1. 国际化方案：轻量自实现 i18n Context

采用 SolidJS Context + Signal 实现轻量 i18n，不引入第三方库：

- 创建 `src/i18n/` 目录，包含：
  - `context.tsx`：I18nProvider 组件 + useI18n hook，内部维护 locale signal
  - `locales/zh.ts`：中文翻译字典
  - `locales/en.ts`：英文翻译字典
  - `index.ts`：统一导出

- 翻译 key 设计：按页面/组件分组，如 `nav.concept`、`hero.title`、`demo.exportClip`

- 语言切换 UI：在 Nav 组件中添加简洁的中/EN 切换按钮
- 默认语言：根据 `navigator.language` 自动检测，fallback 到英文
- 语言偏好持久化：存入 `localStorage` key `dimcut-locale`

### 2. 文案优化

- 移除中英混排模式，根据当前语言仅显示对应文案
- Hero 主 CTA 按钮：
  - 中文："立即体验"
  - 英文："Try it now"
- 次要按钮：
  - 中文："了解理念"
  - 英文："Read the rationale"
- 所有英文文案保持原意，使用更自然的英语表达

### 3. 修复 Demo 横向滚动条

- 在 `<section id="demo">` 外层或 DemoSection 根元素添加 `overflow-hidden`
- 保持 demo 内部 `overflow-x-auto` 允许用户在窄屏时滚动查看 demo
- 确保 demo 的横向滚动不泄漏到页面级别
