## 背景

## 需求

- 在 DEMO 中实现一个轻量简易版的素材管理
- 添加一个素材管理按钮，点击弹窗展示在轨的素材列表
- 以卡片形式展示素材的关键信息，名字、时长、类型等
- 用户可以删除素材、用户可以从本地导入素材文件
- 主轨必须是视频或者音频，以及ASR文件，是特殊类型素材
  - 如果被删除，则显示占位元素
  - 用户可以替换主轨素材

## 现状分析

### 数据流

- `src/demo/App.tsx` 使用 `createResource(loadDemoData)` 一次性加载演示数据，返回 `IChangeEventData`
- `IChangeEventData` 包含 `mainTrackConf`（主轨配置：视频/音频 + ASR）和 `items`（叠加素材列表）
- `Timeline` 组件接收 `initData` prop，内部通过 `createSignal` 拷贝 `items` 进行独立管理
- `mainTrackConf.item` 从 props 直接读取，`items` 在 Timeline 内部可拖拽修改
- `onChange` 回调将内部状态（items + deletedRanges）同步回 App 层

### 关键类型

- `IMainTrackConf`：`{ item: IVideoItem | IAudioItem, asrData?: AsrData }`
- `Item` = `IVideoItem | IAudioItem | IImageItem | ITextItem`
- 所有 `IMediaItem` 均包含 `file: File`

### 约束

- 当前 `initData` 通过 `createResource` 生成，属于只读资源，不支持动态更新
- Timeline 内部状态（items）与 App 层不直接共享，仅通过 `onChange` 单向同步
- 主轨素材被删除后，Timeline 仍需要有效的 `mainTrackConf` 才能渲染

## 技术实现方案

### 整体思路

将素材数据提升到 App 层管理，AssetManager 作为独立弹窗组件读写 App 层状态。通过 `key` 属性强制 Timeline 在数据变更时重新挂载。

### 1. 数据状态提升

- App.tsx 将 `createResource(loadDemoData)` 改为 `createSignal<IChangeEventData | null>(null)`，初始化时 `onMount` 调用 `loadDemoData()` 写入 signal
- 新增 `mainTrackDeleted` signal（boolean）：标记主轨是否被软删除，删除后 Timeline/PreviewPlayer 显示空白
- 新增 `asrDeleted` signal（boolean）：标记 ASR 是否被独立删除（ASR 可独立管理）
- 新增 `assetVersion` signal（number）：作为 Timeline 的 `key`，每次素材变更递增触发重新挂载
- 新增 `showAssetsModal` signal（boolean）：控制素材管理弹窗显隐

### 2. 主轨删除 → 空白渲染

- 删除主轨时设置 `mainTrackDeleted = true`
- App 层条件渲染：当 `mainTrackDeleted` 为 true 时，Timeline 区域显示空白占位，PreviewPlayer 区域也显示空白占位
- 替换主轨时清除 `mainTrackDeleted`，更新 `mainTrackConf.item`，递增 `assetVersion`

### 3. ASR 独立管理

- ASR 作为独立素材卡片展示在弹窗的「主轨素材」区域
- 删除 ASR：设置 `asrDeleted = true`，更新 `mainTrackConf` 清除 `asrData`，递增 `assetVersion`
- 导入 ASR：触发 `<input type="file" accept=".json">`，解析为 `AsrData`，更新 `mainTrackConf.asrData`，清除 `asrDeleted`，递增 `assetVersion`
- ASR 删除后在弹窗中显示占位卡片（"ASR 已移除"），可重新导入

### 4. AssetManagerModal 组件

新建 `src/demo/AssetManagerModal.tsx`：

- Props：`data`、`mainTrackDeleted`、`asrDeleted`、`onReplaceMainTrack`、`onDeleteMainTrack`、`onDeleteAsr`、`onImportAsr`、`onDeleteItem`、`onImportFiles`、`onClose`
- 固定定位弹窗，背景半透明遮罩，居中显示，宽度 480px
- 内部分为「主轨素材」区域（含主轨卡片 + ASR 卡片）和「叠加素材」区域
- 底部「导入素材」按钮

### 5. 素材卡片

每张卡片展示：

- 文件名（`file.name` 或占位文本）
- 时长（格式化 `endTime - startTime`）
- 类型标签（video / audio / image / text / asr）
- 操作按钮（删除 / 替换 / 导入）

主轨卡片特殊处理：

- 删除后变为占位卡片（灰色背景 + "主轨素材已移除" + 「替换素材」按钮）
- 替换按钮触发 `<input type="file" accept="video/*,audio/*">`

ASR 卡片特殊处理：

- 删除后变为占位卡片（"ASR 已移除" + 「导入 ASR」按钮）
- 导入按钮触发 `<input type="file" accept=".json">`

### 6. 叠加素材导入/删除

- 导入：`<input type="file" multiple accept="video/*,audio/*,image/*">`
- 视频/音频：通过 `<video>/<audio>` 元素获取时长
- 图片：固定默认时长 5 秒
- startTime = 0，不需要自动排列
- 删除：从 `data.items` 移除，递增 `assetVersion`

### 7. 素材管理按钮

- 在 App.tsx 工具栏（导出按钮旁）添加「素材管理」按钮
- 点击切换 `showAssetsModal`

## bugs

### Bug 1：ASR 素材也需要支持替换/删除

**现状**：

- `AssetManagerModal.tsx` L192-197：ASR 正常状态卡片只有"删除"按钮，缺少"替换"按钮
- ASR 删除后（L158-175）显示占位卡片，只有"导入 ASR"按钮

**修复方案**：

- ASR 正常状态卡片增加"替换"按钮，与"删除"按钮并排
- 替换按钮点击触发 ASR 文件选择器（复用现有 `asrFileInput`），选择文件后调用现有 `onImportAsr` 回调（功能一致，无需新增 prop）
- ASR 卡片内容改为显示：文件名 + ASR 总时长（基于 segments 最后一个 segment 的 endTime）
- 需要存储 ASR 文件名：在 `AsrData` 类型中新增可选字段 `filename?: string`，导入时写入 `file.name`
- ASR 占位卡片中"导入 ASR"按钮保持不变

### Bug 2：素材类型标签颜色使用 ITEM_COLORS 常量

**现状**：

- `AssetManagerModal.tsx` L26-32 使用了自定义 tailwind 类名（`bg-blue-100 text-blue-700` 等）作为标签颜色
- `src/lib/constants.ts` 定义了 `ITEM_COLORS`：`video: "#FF8702"`, `audio: "#FFCEE1"`, `image: "#B7E5D2"`, `text: "#C9F0FF"`

**修复方案**：

- 删除 `TYPE_COLORS` 常量（tailwind 类名），改为从 `ITEM_COLORS` 导入色值
- 在 `constants.ts` 的 `ITEM_COLORS` 中新增 `asr: "#FFE0B2"`
- 标签使用 `style={{ backgroundColor: ITEM_COLORS[type] }}` 设置背景色
- 标签文字颜色统一使用深色 `text-gray-700`
