## 背景

### Rethinking the Video Editing Timeline: A Multi-Line, Text-First Approach

I've been exploring a different interaction model for video/podcast editing timelines, and I'd love feedback from people who actually edit regularly.

The idea
Instead of the traditional single horizontal timeline, I'm experimenting with a multi-line, text-first timeline:

Break the 1D timeline into multiple rows
→ Reduces excessive horizontal scrolling
→ Feels more like reading than scrubbing

Allow non-overlapping clips to share the same track (even across types)
→ Video, audio, SFX, images can coexist if they don't overlap
→ Increases information density without adding more tracks

Make transcript (ASR) the primary navigation layer
→ Especially for knowledge content / podcasts
→ Faster to locate, edit, and restructure via text instead of waveform

Why this direction?
For a lot of content (talking-head videos, podcasts, educational content), editing is less about frame-perfect visual timing and more about:

finding segments

removing / rearranging ideas

aligning supporting media to speech

So I'm wondering if timeline-as-text (with media attached) might be a better default than waveform-first editing.

## 现状分析

### 入口与渲染链路

- `index.html` 是 Vite 的标准入口，挂载 `<div id="root">` 并加载 `src/index.tsx`。
- `src/index.tsx` 直接 `render(() => <App />, root)`，其中 `App` 来自 `src/demo/App.tsx`。
- `App` 组件目前以单一 `<section>` 形式渲染了 demo 全部交互（时间轴 + 预览播放器 + 摘录片段 + 素材管理弹窗）。
- 容器写死 `min-w-[1400px] max-w-[1400px] mx-auto p-4`，宽度刚性约束，意味着 demo 在小于 1400px 的视口里会出现横向滚动条。

### 视觉与风格基线

- 全局样式 `src/index.css` 仅 `@import "tailwindcss"` + `body { margin: 0 }`，没有自定义字体、配色、CSS 变量。
- `index.html` 没有引入字体或 favicon 文案。
- 现 demo 风格：白底、黑色按钮、灰色边框，朴素的工程化界面，没有"艺术范、打破常规"的视觉表达。

### 已删除的旧 landing.html

- `public/landing.html` 在 `c3c15c5 feat: landing page` 中曾被引入，本次工作树状态中已被 `D`（删除）。
- 旧版本是深色风格、独立静态 HTML（与 SolidJS demo 解耦），与本次需求"浅色 + 整合 demo"方向不符，因此已删除符合预期。
- 旧版的 timeline 视觉静态 mock 可作为风格参考之一（多行折叠、ASR 高亮删除线、播放头红线），但配色、整体语言需要重做。

### 项目特色（用于落地页文案凝练）

- 多行折行时间轴（不再单一长条横向滚动）
- 不同类型片段在不重叠时可共用同一行（信息密度更高）
- ASR 文本作为主导航层，编辑节目/播客像编辑文档一样
- 删除区间可视化、跳过播放
- 视频导出、片段摘录导出
- 技术栈 SolidJS + mediabunny，全本地处理（隐含的"无上传"卖点）

### 技术约束

- 项目以 Vite Library Mode 构建（`vite.config.ts` 输出 `src/lib/index.ts`），但 dev/preview 使用 demo 入口，落地页应作为 dev/preview 入口的一部分，不进入 lib 产物。
- 依赖：`solid-js`、`mediabunny`、Tailwind v4。无路由库（`solid-app-router` 等）。
- 没有图片资源支撑落地页（仅 `src/assets/hero.png`、`solid.svg`、`vite.svg`），需要靠 CSS/SVG 自绘视觉。

## 技术实现方案

### 1. 目录与文件结构

新增 `src/landing/` 目录，承载落地页相关代码：

```
src/landing/
├── LandingApp.tsx          # 整合入口：Nav + Hero + Concept + DemoSection + Footer
├── Nav.tsx                 # 顶部固定导航（Logo / 锚链接 / GitHub）
├── Hero.tsx                # 首屏：双语主张 + CTA + 时间轴示意
├── TimelineIllustration.tsx# 静态 SVG/JSX 时间轴示意（Hero 视觉主体）
├── Concept.tsx             # 第二屏：核心理念三段（中英对照）
├── DemoSection.tsx         # demo 包装层：标题、引导文案、保留 <App />
└── Footer.tsx              # 页脚：版权 + 作者风痕 + GitHub
```

`src/demo/App.tsx` 保持不变，仅修改 `src/index.tsx` 改为渲染 `LandingApp`。

### 2. 整合方式

- `LandingApp` 是一个纯 SolidJS 组件，垂直拼接：`<Nav />` → `<Hero />` → `<Concept />` → `<DemoSection>` → `<Footer />`。
- `<DemoSection>` 给 demo 区域加一个浅色背景容器 + 区块小标题 + 简短引导文案，**不修改 demo 内部任何交互代码**，保证现有功能完整保留。
- demo 容器外层（`src/demo/App.tsx` 内的 `min-w-[1400px]`）保持不变 — 已确认接受其在 < 1400px 视口横向滚动；landing 区块自身使用响应式宽度（容器 `max-w-[1100px]`）。

### 3. 视觉风格语言（浅色 / 艺术 / 打破常规）

确定风格基调，作为后续编码的设计 token：

| Token | 值 | 用途 |
|-------|-----|------|
| 背景 | `#fafaf7`（暖白米色） | 整体底色，与纯白拉开层次 |
| Demo 区背景 | `#f4f4f0`（更冷一点的浅灰米色） | 让 demo 像"框景"嵌入页面 |
| 主前景 | `#0a0a0a` | 标题与重点正文 |
| 次前景 | `#6b6b6b` | 解释性文字 |
| 极弱前景 | `#bdbdbd` | 装饰文字、刻度、罗马数字 |
| 强调色 | `#c0392b`（与 demo Playhead/Deleted 红同色系） | 关键词、播放头、删除区间，与 demo 视觉串联 |
| 衬线字体栈 | `ui-serif, "Songti SC", "Noto Serif SC", "Source Han Serif SC", "Source Han Serif CN", Georgia, Cambria, "Times New Roman", serif` | Hero/章节大标题（中英共用，系统衬线优先） |
| 无衬线字体栈 | `ui-sans-serif, system-ui, -apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", Helvetica, Arial, sans-serif` | 正文、UI、英文小标 |
| 装饰元素 | 大号衬线斜体词、细分割线、轻微纸纹背景 | "打破常规"通过排版而非滤镜实现 |

字体策略：**不使用任何外链 Web 字体**，全部依赖系统衬线/无衬线字体降级，体现"克制、本地、低依赖"的产品调性。

"打破常规"的具体手段：
- Hero 不居中堆叠 — 采用左对齐大字 + 右侧时间轴示意的不对称构图。
- 用大号衬线斜体的关键词（如 *script*）做视觉锚点。
- 章节编号用罗马数字 `I / II / III` 的极小灰字作为段落标记，类似杂志排版。
- 文本与时间轴示意之间用细横线（hairline）做分割，避免色块。
- 整页保留大量留白（每屏 padding 至少 6–8rem 上下）。

### 4. 双语策略（中英并列，无切换器）

主张让双语共存即设计的一部分（也是"打破常规"的一种排版手段），不引入语言切换器：

- **Hero 主标题**：英文大号衬线为主视觉，中文衬线副标题作为对照行。
  - 英：`Edit your timeline like reading a *script*.`
  - 中：`把时间轴当作剧本来读。`
- **Hero 副说明**：英文一句话解释 + 中文一句话对照。
- **Concept 三段论**：每段 = 罗马数字 + 英文斜体短论点 + 中文短论点 + 中性灰展开说明（中英各一句）。
- **DemoSection 标题**：`Live demo / 在线体验`，下方一行隐私说明（中文为主）。
- **Nav / CTA / Footer**：英文为主，避免重复。

### 5. Nav 设计要点

- 固定在视口顶部，毛玻璃 `backdrop-blur` + `rgba(250,250,247,0.78)` 半透明底；底部 hairline。
- 左：Logo 文字 `DimCut`（衬线粗体）。
- 右：锚链接 `Concept` `Demo`、外链 `GitHub`（→ `https://github.com/hughfenghen/dimcut`，`target="_blank" rel="noopener noreferrer"`）。

### 6. Hero 区设计要点（第一屏，min-h-screen）

- 网格布局：左侧（≈ 7/12 列）放标题/副标题/CTA；右侧（≈ 5/12 列）放 `<TimelineIllustration />`。窗口宽度 < 960px 时降级为单列（标题在上，示意在下）。
- 标题：左对齐，英文大字（`clamp(3rem, 6vw, 5.2rem)`，衬线，行高 1.05），其中 `*script*` 用衬线斜体 + 强调色 `#c0392b`。
- 中文标题副行：紧贴英文标题下方，衬线，字号约 1.4rem，颜色为次前景。
- 副段落：英文一句 + 中文一句，灰色，`max-w 480px`。
- CTA：主按钮 `Try the demo ↓`（锚点 `#demo`，黑底白字，方角细边）+ 幽灵按钮 `Read the rationale →`（锚点 `#concept`，描边）。
- 角落标签：`I.` 极小灰字浮于 Hero 左上角作为章节标记，呼应杂志感。

### 7. TimelineIllustration 设计要点

- 一个浅色卡片（白底 / 极淡阴影 / 1px hairline），内部静态拼装：
  - 顶栏 mac 风格三色点（颜色压低饱和度，避免儿童化）。
  - 顶部刻度：00:00 / 00:08 / 00:16 / 00:24（极小灰字）。
  - 三行 timeline，行间 hairline 分割：第 1 行 video clip + 嵌入波形条；第 2 行 audio clip 共行的 image clip；第 3 行又一个 video clip + ASR 文字行（带高亮 + 删除线）。
  - 一段红色透明 deleted range 横跨第 2 行某段。
  - 一条红色细 playhead 竖线，顶部圆点。
- 全部用 div + Tailwind + 几个内联样式实现，不依赖外部 SVG。

### 8. Concept 区设计要点（第二屏 ≈ 0.5–1 屏，id="concept"）

- 不使用三栏 feature card 这种俗套布局。
- 三段交错排版的"宣言式"段落，水平略微错位（第 1 段左对齐，第 2 段缩进 12%，第 3 段缩进 24%，制造阅读流的"阶梯"），每段：
  - 罗马数字编号（极小灰字 `I.` `II.` `III.`）。
  - 英文斜体短论点（衬线，约 1.6rem）。
  - 中文短论点（衬线，与英文同字号，颜色为主前景）。
  - 中性灰短展开（无衬线，约 0.95rem，中英各一句）。
- 段间 hairline 分隔，整体节奏类似一篇评论文章而非产品页。

三段文案：
- I. *Break the 1D timeline.* / 把一维时间轴折叠成多行。 — 读图替代横向滚动；像翻页而非拖动。
- II. *Pack clips densely.* / 不重叠的片段共享一行。 — 视频、音频、图像能在不冲突时共栖；信息密度更高。
- III. *Read, don't scrub.* / 用文字定位，而不是用波形。 — ASR 转写作为主导航层；编辑像编辑文档。

### 9. DemoSection 区设计要点（id="demo"）

- 背景过渡到稍冷的浅灰 `#f4f4f0`，让 demo 部分自然成为一个"框景"区块。
- 区块顶部：极小章节标记 `IV.` + 标题 `Live demo / 在线体验` + 一行说明（"All processing happens locally in your browser, no upload. / 所有处理在本地浏览器完成，无需上传。"）。
- 然后直接渲染 `<App />`（来自 `src/demo/App.tsx`），不改动 demo 内部 DOM/样式。
- 外层包裹一个浅色卡片容器（白底 / 圆角 12px / 1px hairline / 极淡阴影 / 大留白），让 demo 视觉上嵌入页面。
- 不引入"快速入门提示"小条带（避免重复 demo 自身已有的底部提示，保留 demo 原行为）。

### 10. Footer 设计要点

- 极简横排，hairline 上分隔。
- 左：`© 2026 DimCut · 风痕`（"风痕" 链接到 `https://fenghen.me`，外链 `target="_blank"`）。
